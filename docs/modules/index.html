This file contains all of the exports that make it into the pure javascript / bundled flat structure.

To create a dag structure from data, a necessary first step before computing a layout, see the three creation algorithms:

  • HierarchyOperator - when the data already has a dag structure.
  • StratifyOperator - when the dag has a tabular structure, referencing parents by id.
  • ConnectOperator - when the dag has a link structure and is specified as pairs of nodes.

To layout a dag see the two dag layout algorithms:

  • SugiyamaOperator - for a general layered dag representation
  • ZherebkoOperator - for a simple topological layout
  • GridOperator - for an alternate topological layout
example

Using d3-dag is usually a two step process, first you choose a creation method for your given data structure, this turns data like data into a dag structure, then you lay it out using one of the layout algorithms. The creation methods follow a similar structure, as do the layout methods. Each can be tweaked with a fluent api.

const creation = dagConnect(); // optionally use fluent interface to customize
const dag = creation(data);
const layout = sugiyama(); // optionally customize with fluent interface
const { width, height } = layout(dag);
for (const node of dag) {
  // dag connect creates data with the node id
  console.log(node.data.id, node.x, node.y);
}
