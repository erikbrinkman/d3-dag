Module sugiyama/utils

Utility types for sugiyama layout

This module should only really matter for those looking to develop custom CoordOperators, DecrossOperators, or in the rare case advanced SugiNodeSizeAccessors.

The NodeDatum used for layered SugiyamaOperator layouts

Nodes in the original graph have a layer and a reference to the original node. "dummy nodes" have a link to the parent and child of the edge their on in the original dag, as well as their actual layer. Given that duplicate edges aren't allowed, this uniquely defines each dummy node.

sugify

  • sugify<N, L>(dag: Dag<N, L>): SugiNode<N, L>[][]
    • Defined in src/sugiyama/utils.ts:79

    Create a dag of layered SugiNodes from a layered dag.

    Internally the sugiyama layout converts a dag into SugiNodes to store metadata about the layout before copying the data back to the original dag. SugiNodes are normal dag nodes who's data have a special relation to the underlying dag's structure. Each new node contain the wrapped nodes for real nodes, or links to the source and target nodes for dummy nodes.

    This method returns the nodes in their appropriate layers. After updating metadata on the layered nodes, call unsugify to copy the data to the underling dag.

    The only reason to use this is if calling parts of the sugiyama chain independently.
