    • Defined in src/sugiyama/decross/index.ts:23

    A decrossing operator that minimizes the number of crossings by looking at every pair of layers.

    This method can be very fast and is a general heuristic for efficient minimization. Customize with a two layer operator with order or use one of the built-in TwolayerOperators.

    This method can also make multiple passes in an attempt to produce a better layout.

    Sets the initialization passes to the specified DecrossOperators and returns a new operator.

    For every initialization operator, this will run the two layer heuristic, ultimately choosing the ordering that minimized overall crossings. For this reason, only quick decrossing operators should be used, not expensive ones. (default: [noop])
